  Filtro obligatorio en todas las consultas

    (identifica la empresa y sucursal INSEVIG dentro del ERP compartido)

  ── Tablas de nómina (solo lectura para consultas) ──────────────────────────
    RPEMPLEA   — empleados: cedula, nombres, apellidos, cargo, sueldo,
                 fecha_ingreso, cod_empleado
    RPHISTOR   — nómina histórica cerrada (~2.5M filas)
                 columnas clave: empleado, clase, valor, fecha_ven
    RPINGDES   — nómina del mes abierto/actual
                 columnas clave: empleado, clase, valor, fecha_ven

    Recomendación: no alterar estas tablas desde módulos RRHH.
    Son mantenidas por el ERP y son fuente de verdad de nómina.

  ── Otras tablas del ERP (lectura y/o escritura según módulo) ───────────────
    Cada módulo que necesite leer o escribir en otras tablas del ERP
    debe documentar explícitamente qué tablas usa y qué operaciones
    realiza (INSERT / UPDATE / DELETE), y validar con el proveedor del
    ERP si la operación es compatible con el sistema.

    Buenas prácticas para escritura en SQL Server:
      - Usar siempre parámetros en las consultas (nunca concatenar strings)
      - Usar transacciones (conn.autocommit = False; conn.commit())
      - Manejar errores con rollback explícito en el except
      - Registrar toda escritura en la tabla de auditoría del módulo
      - Nunca escribir directamente desde hilos secundarios sin commit/rollback

  ── Driver ODBC — orden de prioridad ────────────────────────────────────────
    1. ODBC Driver 17 for SQL Server   ← preferido
    2. ODBC Driver 18 for SQL Server
    3. ODBC Driver 13 for SQL Server
    4. ODBC Driver 11 for SQL Server
    5. SQL Server                      ← driver nativo Windows (fallback)

    La función get_sql_conn() de sync_sqlserver.py prueba estos en orden
    hasta que uno conecta. Si ninguno conecta lanza excepción con mensaje
    descriptivo.

  ── Problema TLS con SQL Server 2008 R2 ─────────────────────────────────────
    El servidor usa TLS 1.0 (obsoleto). Python moderno + OpenSSL rechazan
    esta versión por defecto. Solución: archivo openssl_legacy.cnf que
    habilita TLS 1.0, cargado antes de importar pyodbc:

      import os
      os.environ['OPENSSL_CONF'] = ruta_a_openssl_legacy.cnf

    En EXE frozen:  ruta = bundle_dir()  (= sys._MEIPASS)
    En desarrollo:  ruta = carpeta del script

    Contenido de openssl_legacy.cnf:
      [openssl_init]
      ssl_conf = ssl_sect
      [ssl_sect]
      system_default = system_default_sect
      [system_default_sect]
      MinProtocol = TLSv1
      CipherString = DEFAULT@SECLEVEL=0

    Este archivo DEBE incluirse en el .spec de PyInstaller (sección datas)
    para que funcione en el EXE.

  ── Código Python de conexión ────────────────────────────────────────────────
    SQL_CFG = {
        'driver':   'ODBC Driver 17 for SQL Server',
        'server':   '192.168.2.115',
        'database': 'insevig',
        'uid':      'sa',
        'pwd':      'puntosoft123*',
    }

    Módulo de referencia: sync_sqlserver.py
      get_sql_conn()                          → pyodbc.Connection
      get_movimientos_empleado(cod, fi, ff)   → rows RPHISTOR + RPINGDES
      probar_conexion()                       → True / False

    Patrón recomendado para escritura:
      conn = get_sql_conn()
      conn.autocommit = False
      try:
          c = conn.cursor()
          c.execute("UPDATE tabla SET campo=? WHERE id=?", (valor, id))
          conn.commit()
      except Exception:
          conn.rollback()
          raise
      finally:
          conn.close()

4. SQLITE (BASE DE DATOS LOCAL / RED)
================================================================================

  SQLite es la base de datos TRANSACCIONAL de cada módulo.
  Se usa para todos los datos propios del módulo (registros, configuración,
  historial local). No reemplaza a SQL Server ni a Supabase, sino que
  actúa como capa local rápida y siempre disponible (sin red requerida).

  ── Resolución de ruta en 3 pasos (db.py) ───────────────────────────────────
    1. config.json → "db_red_custom"
       (el usuario puede configurar una ruta personalizada)
    2. Ruta en red roberto-pc:
       \\Roberto-pc\rrhh2021\ARCHIVOS ASISTENTE AOC\1.TURNOS\data\<modulo>.db
       En Linux: /mnt/roberto-pc/ARCHIVOS ASISTENTE AOC/1.TURNOS/data/<modulo>.db
    3. Carpeta junto al exe: <app_dir()>\<modulo>.db  (fallback local)

    La base en roberto-pc es la fuente de verdad compartida entre equipos.
    Todos los puestos que corran el programa usan la MISMA base de red.

  ── Tabla compartida entre módulos (cuando se unifiquen) ─────────────────────
    La tabla 'empleados' (caché de RPEMPLEA) se comparte entre módulos.
    Cada módulo tiene sus propias tablas pero apuntan a la misma db.

  ── Escritura en SQLite ──────────────────────────────────────────────────────
    SQLite permite lectura y escritura completa.
    Usar siempre con context manager para commits automáticos:
      with get_connection() as conn:
          conn.execute("INSERT INTO tabla (...) VALUES (?,...)", params)
    Para lecturas sin escritura: conn.execute() directo está bien.

  ── Módulos Python base ──────────────────────────────────────────────────────
    db.py      → get_connection(), create_tables(), migrate()
    model.py   → CRUD del módulo (único punto de acceso a datos)

9. PYINSTALLER — NOTAS IMPORTANTES
================================================================================

  El EXE extrae todos sus archivos a una carpeta temporal (sys._MEIPASS)
  al arrancar. Hay dos tipos de rutas:

  ── Archivos de solo lectura (templates, imágenes, configs bundleados) ───────
    from _paths import bundle_dir
    ruta = os.path.join(bundle_dir(), 'nombre_archivo')

    bundle_dir() = sys._MEIPASS  en el EXE frozen
    bundle_dir() = dirname(__file__)  en desarrollo

  ── Archivos de escritura (db, logs, config.json, documentos generados) ─────
    from _paths import app_dir
    ruta = os.path.join(app_dir(), 'nombre_archivo')

    app_dir() = dirname(sys.executable)  en el EXE frozen
    app_dir() = dirname(__file__)  en desarrollo

  ── Regla de oro ─────────────────────────────────────────────────────────────
    NUNCA usar __file__ o rutas relativas directamente.
    SIEMPRE usar bundle_dir() o app_dir() de _paths.py.

  ── config.json ──────────────────────────────────────────────────────────────
    No se bundlea en el EXE. Al primer arranque se crean los defaults
    definidos en app_config._DEFAULTS y se guardan junto al exe.
    En arranques siguientes se carga desde ahí.

  ── Splash screen ────────────────────────────────────────────────────────────
    splash.png debe existir en la raíz del proyecto.
    En app.py, después de crear la ventana principal:
      try:
          import pyi_splash
          pyi_splash.close()
      except Exception:
          pass

  ── supabase.yaml NO se bundlea ──────────────────────────────────────────────
    El archivo config/supabase.yaml debe estar junto al exe en producción
    (no dentro del bundle). Copiarlo manualmente al desplegar si el módulo
    usa Supabase.
